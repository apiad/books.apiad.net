# Mostly Harmless Books Storefront - Design Document

## 1. Architectural Overview

A zero-dependency, client-side Single Page Application (SPA) built with vanilla JavaScript. Uses a JSON data structure for catalog metadata and renders HTML dynamically.

- **Hosting:** GitHub Pages (`books.apiad.net`)
- **Styling:** Tailwind CSS via CDN
- **State:** Immutable, loaded once at page load
- **Payment:** Gumroad overlay checkout
- **Book Content:** Served from `/books/{bookId}/` subdirectories (generated by Quarto)

## 2. Directory Structure

```
/
├── index.html          # Main SPA with embedded JS rendering
├── design.md           # This document
├── books/              # Individual book websites (Quarto output)
│   ├── tsoc/           # The Science of Computation
│   ├── mhai/           # Mostly Harmless AI
│   ├── chatbots/       # How to Train your Chatbot
│   └── graphs/         # Mostly Harmless Graphs (future)
└── CNAME               # books.apiad.net
```

## 3. Data Model

### JSON Schema (`catalogData`)

```typescript
interface Catalog {
  compendium: Compendium;
  categories: Category[];
}

interface Compendium {
  id: string;
  title: string;
  description: string;
  price: number;
  gumroadUrl: string;
}

interface Category {
  id: string;
  name: string;      // e.g., "📚 Early Access"
  description: string;
  items: Book[];
}

interface Book {
  id: string;        // e.g., "tsoc", "mhai"
  title: string;
  audience: string;  // e.g., "😄 General"
  description: string;
  status: string;    // "Alpha" | "Beta" | "Delta" | "Epsilon" | "Omega"
  statusEmoji: string; // "🔴" | "🟠" | "🔵" | "🟣" | "🟢"
  progress: number;  // 0-100 percentage
  pages: number;     // Current page count
  targetPages: number; // Estimated total (optional, for display)
  price: number | null; // null = Coming Soon
  gumroadUrl: string | null;
  readUrl: string;    // e.g., "/books/tsoc"
  coverUrl: string;   // Path to cover image
}
```

## 4. UI/UX Design

### Color Palette

- **Background:** `#0f0f0f` (near black)
- **Card Background:** `#1a1a1a` (dark gray)
- **Card Hover:** `#222222` (lighter gray)
- **Primary Accent:** `#f97316` (orange-500)
- **Secondary Accent:** `#8b5cf6` (violet-500)
- **Success/Complete:** `#22c55e` (green-500)
- **Text Primary:** `#fafafa` (near white)
- **Text Secondary:** `#a1a1aa` (zinc-400)
- **Border:** `#27272a` (zinc-800)

### Typography

- **Headings:** "Playfair Display" (serif, elegant)
- **Body/UI:** "Inter" (sans-serif, clean)
- **Emojis:** Native system emojis

### Progress Bar Design

Cute gradient progress bars with smooth transitions:

```
┌─────────────────────────────────────────┐
│ ████████░░░░░░░░░░░░░░░░░░░░░░░░░░░░░ │
└─────────────────────────────────────────┘
```

- **Height:** 8px with rounded-full
- **Background:** zinc-800
- **Fill:** Linear gradient from orange-500 to violet-500
- **Animation:** 0.3s ease transition on width change
- **Percentage label:** Small text showing X%

### Card Design

- **Border radius:** `rounded-2xl` (16px)
- **Shadow:** `shadow-lg shadow-black/20`
- **Padding:** `p-6`
- **Border:** 1px zinc-800, hover: orange-500/30
- **Transition:** 0.2s ease all

### Button Design

**Primary (Buy):**
- Background: orange-500
- Text: white, font-semibold
- Hover: orange-600, scale-105
- Rounded: `rounded-xl`

**Secondary (Read Now):**
- Background: transparent with violet-500 border
- Text: violet-400
- Hover: violet-500 background, white text
- Icon: 📖 before text

**Disabled (Coming Soon):**
- Background: zinc-800
- Text: zinc-500
- Cursor: not-allowed

### Responsive Breakpoints

- **Mobile (<640px):** 1 column grid
- **Tablet (640-1024px):** 2 column grid
- **Desktop (>1024px):** 3-4 column grid
- **Max content width:** `max-w-6xl` centered

## 5. Component Functions

### `renderLayout(catalogData)` → string

- HTML shell with Tailwind CDN
- Google Fonts (Playfair Display, Inter)
- Gumroad script tag
- Dark mode by default
- Calls other render functions and injects into `#catalog-root`

### `renderCompendium(compendium)` → string

- Prominent hero card at top
- Gradient border (orange → violet)
- "Best Value" badge
- Title, description
- Price formatted as $XX.XX
- Two buttons: "Buy Bundle" (Gumroad), "Learn More" (anchor to info)

### `renderCategory(category)` → string

- Category name as h2 with emoji
- Description paragraph
- Grid of book cards (responsive columns)

### `renderBookCard(book)` → string

**If priced (Active):**
- Cover image (or placeholder gradient)
- Title (h3)
- Audience badge
- Status badge + emoji
- Progress bar with percentage
- Page count: "XX of YY pages"
- Two buttons: "Read Now" (primary), "Buy on Gumroad" (secondary)

**If Planned (No price):**
- Dimmed cover or icon
- Title, audience, description
- Status badge: "🎯 Planned"
- No buttons, just "Coming Eventually" text

### `renderProgressBar(progress, statusEmoji)` → string

```html
<div class="flex items-center gap-3">
  <div class="flex-1 h-2 bg-zinc-800 rounded-full overflow-hidden">
    <div class="h-full bg-gradient-to-r from-orange-500 to-violet-500 rounded-full transition-all duration-300" 
         style="width: {progress}%"></div>
  </div>
  <span class="text-xs text-zinc-400 font-mono">{progress}%</span>
</div>
```

### `renderButton(url, text, variant, icon)` → string

- variant: "primary" | "secondary" | "disabled"
- icon: emoji string to prepend
- data-gumroad-overlay-checkout="true" for primary links

## 6. Rendering Pipeline

1. Browser loads `index.html`
2. Tailwind CDN and Gumroad script load async
3. `DOMContentLoaded` fires
4. Inline JS parses `catalogData`
5. Generates HTML via component functions
6. Injects into `#catalog-root` via `innerHTML`
7. Browser paints UI

## 7. Migration from index.qmd

**Content to extract:**
- Compendium link → compendium object
- Early Access category → first Category object
- Book details (title, audience, status, progress, pages, links) → Book objects
- Backburner category → second Category object
- Backburner book descriptions → Book objects (no price)

**Content to discard:**
- Legend text (moved to design doc or tooltip)
- Individual book full descriptions (already in book websites)

**Automated extraction:** Create a Node.js script to parse `index.qmd` and generate `catalogData` JSON for easy maintenance.
